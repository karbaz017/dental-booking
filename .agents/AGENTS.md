# DentalCare Booking Rules

## Next.js (App Router) Guidelines
*   This project runs **Next.js 16 (React 19)**. Note the rules and deprecation notices under `node_modules/next/dist/docs/` if available.
*   Always use typescript and leverage React server components (RSC) for data fetching and layout management wherever possible.
*   Client components should only be used when interactive UI logic is required (e.g. clicks, state, hooks, or context providers). Place `"use client"` at the top of these files.

## Database & Prisma Guidelines
*   **Prisma Client version**: `6.19.0`
*   **Database**: PostgreSQL
*   All models mapped to database tables should use lower_case snake_case formatting (e.g. `@@map("users")`).
*   **User Roles**: Must adhere to the `Role` enum (`PATIENT`, `FRONT_DESK`, `SUPERVISOR`, `DOCTOR`).
*   **Cascade Deletion**: Cascading deletes are enabled for child entities like `accounts`, `sessions`, `family_members`, `insurances`, and `appointment_reminders`. Do not delete `User` records directly in production workflows unless required.
*   **Chat Pair Keys**: When creating a chat thread, the `pairKey` MUST be generated by sorting the two participant user IDs lexicographically and joining them with `|` (e.g. `user_id_1|user_id_2` where `user_id_1 < user_id_2`). This guarantees a single, unique thread between any two users.

## Tailwind CSS & Styling Rules
*   This project uses **Tailwind CSS v4** with `@tailwindcss/postcss`. Use utility classes in combination with the custom theme variables defined in [globals.css](file:///Users/arbaz/Projects/dental-booking/src/app/globals.css) (such as `--background`, `--foreground`, and `--teal-600`).
*   Avoid adding custom style block overrides unless absolutely necessary.
